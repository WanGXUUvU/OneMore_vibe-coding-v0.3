## D-001 前端实现形态
- 备选：
  - 原生多页面静态站点（HTML + CSS + JavaScript）
  - React / Next.js 单页或多页应用
- 选择：
  - 原生多页面静态站点（HTML + CSS + JavaScript）
- 原因：
  - 目标是验证最小流程，不需要框架复杂度
  - 要满足“浏览器可直接打开”的约束，静态文件最稳妥
  - 能把 `TASK-001` 控制在少量文件内
- 对当前阶段的影响：
  - 首轮任务可以定义为 Bootstrap Task
  - 不需要构建链和包管理器即可完成最小交付

## D-002 文章内容存放方式
- 备选：
  - 每篇文章单独写成静态 HTML
  - 用本地 JavaScript 数据文件维护文章列表和正文
  - 接入 Markdown 解析或 CMS
- 选择：
  - 用本地 JavaScript 数据文件维护文章列表和正文
- 原因：
  - 既能渲染首页列表，也能支撑详情页，结构更统一
  - 比 CMS 或 Markdown 管线更轻，适合 MVP
  - 后续如果升级为更复杂 blog，也便于迁移数据形态
- 对当前阶段的影响：
  - `TASK-001` 需要包含一个本地文章数据文件
  - 详情页将通过 URL 参数或预设 slug 读取本地数据
